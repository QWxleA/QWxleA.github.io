About resurfacing information from your Daily Notes

  • During the day I run into many interesting topics, tweets, posts on Reddit, thoughts from books, too many to keep track of. I keep everything that I might want to use later, some people are picky, I’m not. Not being picky means I don’t have to make any decisions right now -> which reduces [[friction]].

  • If not tagged, the daily note becomes a black hole where information just disappears. There is just too much information to be useful.

  • Tags help to create buckets of slightly related items. But, Ahrens writes in How to Take Smart Notes that tags “should be chosen carefully and sparsely” (otherwise you move the problem from too many daily notes to too many tags).

  • Don’t use tags as a filing-system, tag with the output, or end-use in mind. When do I want to find this note again, not: this belongs here.

Technical background

  • In logseq tags and links are technically treated the same, but in practical use they are not the same:

    • tags are a system of classification

    • links are an API to a note -> easy linkable nodes

      • Evergreen note titles are like APIs
